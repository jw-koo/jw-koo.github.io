## 핵심 판단

### 지금 자동화하기 가장 좋은 루프 1개

**루프 7: 최종 정적 회귀 검증**이 가장 적합하다. 파일 존재·상대 경로·HTML 구조·내부 링크·CSS 오버플로·viewport별 기본 검사를 읽기 전용으로 반복할 수 있고, 실패 시 코드 자동 수정 없이 `HITL_REQUIRED`로 멈출 수 있다.

### 사람이 반드시 승인해야 하는 지점

- 공개할 경력·프로젝트·기술·링크와 개인정보 범위
- 랜덤 적의 등장 주기·속도·체력 감소·난이도 규칙
- 게임 구현 시작, Git commit/push, GitHub Pages 배포
- 동적 검증 환경에서 Node·로컬 HTTP 실행 권한 부여

### Claude Verifier가 확인해야 할 항목

- 파일 존재, 상대 경로, HTML 구조와 내부 링크
- 375px·768px·1440px 반응형 레이아웃과 가로 오버플로
- JavaScript 문법·런타임 오류와 메뉴 상태
- 지렁이 게임의 키보드·스와이프·먹이·점수·체력·충돌·게임오버
- 로컬 HTTP 응답, GitHub Pages 루트 경로 호환성
- 확인되지 않은 개인 정보, 테스트 삭제·완화, 토큰 노출 여부
- 실행 주체·모델·명령·exit code·오류·관련 파일·fingerprint·최종 상태

### Claude 사용 불가 시 Fallback 방식

Claude CLI가 설치되지 않았거나 로그인·모델 실행이 불가능할 때만 `CODEX_FALLBACK`으로 전환한다. Codex가 동일한 검증 목록을 실행하고, 이유·사용 모델·결과를 `AORR_LOG.md`에 기록한다. Claude가 사용 가능하면 Codex는 검증을 중복 실행하지 않는다.

### 운영 전 필요한 테스트·권한·모니터링

- 테스트: Node 문법 검사, 로컬 HTTP smoke test, 실제 375/768/1440px 브라우저 검사, 키보드·스와이프 회귀, 랜덤 적 반복 플레이
- 권한: 저장소 쓰기, 로컬 HTTP/Node 실행, 모바일 브라우저 또는 에뮬레이터, 최종 단계의 Git push·Pages 확인 권한
- 모니터링: 배포 URL HTTP 상태, 자산 404, JavaScript 콘솔 오류, 게임 시작·종료 오류, 최근 정상 commit과 fingerprint

### 현업 적용 다음 액션 3개

1. Node와 로컬 HTTP 실행을 허용한 격리된 Verifier 실행 환경을 만들고 동일 테스트를 고정한다.
2. 게임 수치와 배포 승인 범위를 확정한 뒤, 게임 코어 루프를 한 원인·한 파일 단위로 시작한다.
3. 보호 브랜치·PR 검증·Pages preview·배포 후 smoke test와 rollback용 정상 commit을 설정한다.
