## 충돌·모호성·의존성 메모

- CR-001은 프로젝트 링크만 교체하며 Contact의 개인 LinkedIn 링크를 삭제하거나 바꾸지 않는다.
- CR-002의 “더 개선”은 기존 모던·세련된 방향 안의 제한된 시각 개선으로 해석한다. 전면 재작성이나 새 프레임워크 추가는 범위 밖이다.
- CR-003의 “아이템”이 기존 음식인지 별도 아이템인지 모호하다. 구현 전 별도 피버 아이템을 추가하는 안을 기본 제안으로 두고 확인한다.
- CR-004의 확정값은 피버 아이템 20~30초마다 등장, 6초 지속, 바이러스 +25점, 피버 중 체력 감소 없음이다.
- 실행 순서는 `CR-001 → CR-002 → CR-003 → CR-004 → 전체 회귀 검증`이며, 각 항목은 Claude 변경 전 검증 → Codex 최소 Act → Claude 동일 검증으로 진행한다.
- 모든 개인정보·경력·프로젝트 사실은 `MEMORY.md`와 사용자가 확인한 자료만 사용한다.
